fix_jeditor_description
=======================

Jira 자동화 규칙의 description 필드에 포함된 jeditor HTML을 순수 텍스트로 변환하는 도구입니다.


배경
----
Jira 자동화 규칙 편집 시 jeditor 에디터가 description 필드에 <table>, <tr>, <td> 등
HTML 태그를 삽입하는 경우가 있습니다.
이 도구는 해당 HTML을 제거하고 가독성 있는 순수 텍스트로 정리합니다.


기능
----
- 전체 자동화 규칙에서 HTML description 자동 감지 (jeditor 클래스 + 일반 HTML 태그 포함)
- HTML 태그 제거 및 HTML 엔티티 디코딩 (&nbsp;, &lt;, &amp; 등)
- 블록 태그 (</tr>, </p>, </li> 등)를 줄바꿈으로 변환하여 텍스트 구조 유지
- 변환 전 원본 JSON 자동 백업 (output/jeditor_description/)
- 백업 기반 원본 복원(Rollback) 지원
- DRY_RUN 모드로 실제 변경 없이 사전 확인 가능
- 429/5xx 오류 시 자동 재시도


요구사항
--------
- Python 3.9 이상 (외부 라이브러리 불필요)
- Atlassian API Token


환경 변수
---------
ATLASSIAN_API_TOKEN     (필수) Atlassian API 토큰
ATLASSIAN_BASE_URL      기본값: https://sg-cf-usertest.atlassian.net
ATLASSIAN_EMAIL         기본값: jsjang@osci.kr
DRY_RUN                 기본값: false  (true이면 실제 변경 없이 결과만 출력)
MAX_RETRIES             기본값: 5  (API 오류 시 최대 재시도 횟수)
OUTPUT_DIR              기본값: ./output  (백업 파일 저장 디렉토리)


사용법
------

[1단계] API 토큰 설정

    export ATLASSIAN_API_TOKEN=your-api-token

[2단계] 실행 (기본: DRY_RUN 모드 — 실제 변경 없음)

    python3 fix_jeditor_description.py

실행하면 아래 메뉴가 나타납니다.

    작업을 선택하세요:
      1. HTML → TEXT 변환 (Fix)
      2. 원본 복원 (Rollback) — output/jeditor_description/ 의 백업 JSON 기준
    >>>

[3단계] 실제 변환 적용

    DRY_RUN=false python3 fix_jeditor_description.py

    - DRY_RUN=false 설정 시 실행 전 'yes' 입력으로 최종 확인합니다.
    - 변환 전 원본 JSON이 output/jeditor_description/ 에 자동으로 백업됩니다.

[4단계] 롤백 (원본 복원)

    메뉴에서 2 를 선택하면 백업 파일 기준으로 원본 규칙을 복원합니다.

    DRY_RUN=false python3 fix_jeditor_description.py
    >>> 2


출력 파일
---------
output/
└── jeditor_description/
    ├── 12345_이슈_생성_알림_규칙.json   ← Fix 전 원본 (Rollback용)
    └── ...


출력 예시
---------
Base URL : https://sg-cf-usertest.atlassian.net
Dry Run  : false

cloudId: xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx
전체 자동화 규칙 조회 중...
전체 규칙: 42개
jeditor HTML description 감지: 3개 / 전체 42개

[1/3] [12345] 이슈 생성 알림 규칙
  변환: value.value: 512 chars → 48 chars
  백업 저장: 12345_이슈_생성_알림_규칙.json
  업데이트 완료

변환 완료: 3/3개 성공, 0개 실패
원본 백업 위치: /path/to/output/jeditor_description


주의사항
--------
[주의] DRY_RUN=false 설정 시 실제 Jira 데이터가 변경됩니다.
       반드시 사전에 DRY_RUN 모드로 결과를 확인하세요.

- 롤백은 output/jeditor_description/ 의 백업 파일에 의존합니다.
  Fix를 실행하지 않아 백업이 없으면 롤백할 수 없습니다.
- Jira는 규칙 PUT 후 내부 컴포넌트 ID를 재생성하므로,
  롤백 시 불필요한 ID 필드는 자동으로 제거됩니다.
